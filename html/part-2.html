Asynchronous JavaScript Practice: Part 2

Based on the How to use promises tutorial from MDN Web Docs.

  • Pending: the promise has been created, and the asynchronous function it's associated with has not succeeded or failed yet.
  • Fulfilled: the asynchronous function has succeeded. When a promise is fulfilled, its then() handler is called.
  • Rejected: the asynchronous function has failed. When a promise is rejected, its catch() handler is called.
  • "Succeeded" or "Failed" is based on the API in use. Even if the server responds with a 404 error, the server technically still responded, so the function did succeed.
  • The term settled is used to cover both terms for fulfilled and rejected.
